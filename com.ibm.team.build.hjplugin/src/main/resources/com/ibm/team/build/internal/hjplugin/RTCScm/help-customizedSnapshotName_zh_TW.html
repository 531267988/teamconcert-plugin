針對接受變更之後所建立的 Snapshot，指定其給定名稱。

這個欄位可以設為使用者指定的任何字串，並參照環境變數和建置參數。建議至少包含一個變數，且其值在每一項建置中都是唯一的，例如 BUILD_NUMBER、BUILD_ID 或 BUILD_URL，以確保連續建置是以不同名值來建立 Snapshot。如果這個欄位空白，會改用 Snapshot 預設名稱。若為使用「建置定義」的工作，Snapshot 名稱預設為 "${buildDefinitionId}_#${BUILD_NUMBER}"。 若為使用「串流」或「儲存庫工作區」的工作，Snapshot 名稱預設為 "${JOB_NAME}_#${BUILD_NUMBER}"。

例如，對於名為 "JUnit Integration Build" 的工作，如果將這個欄位設為 "${JOB_NAME}_#${BUILD_NUMBER}_Snapshot"，第一項建置所建立的 Snapshot 名稱會是 "JUnit Integration Build_#1_Snapshot"。
