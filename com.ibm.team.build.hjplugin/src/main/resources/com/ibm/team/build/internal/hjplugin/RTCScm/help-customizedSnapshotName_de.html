Gibt den Namen für die Momentaufnahme an, der nach dem Akzeptieren der Änderungen erstellt wird.

Für dieses Feld kann eine beliebige benutzerdefinierte Zeichenfolge festgelegt werden, die auf Umgebungsvariablen und Buildparameter verweist. Es wird empfohlen, mindestens eine Variable einzubeziehen, deren Wert eindeutig für jeden Build ist, wie z. B. BUILD_NUMBER, BUILD_ID oder BUILD_URL, um sicherzustellen, dass aufeinander folgende Builds Momentaufnahmen mit unterschiedlichen Namen erstellen. Wenn dieses Feld leer ist, wird ein Standardname für die Momentaufnahme verwendet. Für Jobs, die eine Builddefinition verwenden, lautet der Standardname der Momentaufnahme "${buildDefinitionId}_#${BUILD_NUMBER}". Für Jobs, die einen Datenstrom oder einen Repository-Arbeitsbereich verwenden, lautet der Standardname der Momentaufnahme "${JOB_NAME}_#${BUILD_NUMBER}".

Beispiel: Für einen Job mit dem Namen "JUnit Integration Build" ist der Name der Momentaufnahme, die durch den ersten Build erstellt wird, "JUnit Integration Build_#1_Snapshot", wenn diese Feld auf "${JOB_NAME}_#${BUILD_NUMBER}_Snapshot" gesetzt wird.
